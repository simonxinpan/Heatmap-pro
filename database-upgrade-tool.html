解决 "column last_price does not exist" 错误

🚨 检测到的问题

错误信息：column "last_price" does not exist

根本原因：Neon数据库中的stocks表缺少缓存字段（last_price, change_amount, change_percent, last_updated）

影响：API无法查询股票报价数据，前端只能显示静态模拟数据

✅ 解决方案

执行ALTER TABLE命令，为stocks表添加缺失的字段。这是一个安全操作，不会影响现有数据。

步骤1: 检查当前表结构

首先检查stocks表的当前结构，确认缺失的字段

步骤2: 执行数据库升级

添加缺失的字段到stocks表

ALTER TABLE stocks ADD COLUMN IF NOT EXISTS last_price NUMERIC(10, 2), ADD COLUMN IF NOT EXISTS change_amount NUMERIC(10, 2), ADD COLUMN IF NOT EXISTS change_percent NUMERIC(8, 4), ADD COLUMN IF NOT EXISTS last_updated TIMESTAMPTZ;
步骤3: 初始化缓存数据

从Finnhub API获取最新报价并填充到新字段

步骤4: 验证修复结果

测试API是否能正常返回完整的股票数据

步骤5: 查看热力图

打开热力图页面，确认显示全部502只股票
